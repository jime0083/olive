# Olion Site - Project Guidelines

## 必須ルール（厳守）

### 実装完了後のデプロイ（最重要）

**✅ デプロイはユーザーの確認を待たずに自動で実行すること（2026-07-13追加）**

- ユーザーは本番環境（https://jime0083.github.io/olive）でデザイン・動作を確認する運用のため、修正が完了したら**確認を求めずに**即座にコミット・プッシュ・デプロイまで実行する。
- 「デプロイしてよいですか？」といった確認は不要。ビルドとTypeScriptチェックが通ったら、そのままデプロイする。

**実装作業が完了したら、必ず以下の手順でGitHub Pagesにデプロイすること：**

```bash
# 1. 変更をコミット
git add .
git commit -m "<type>: <description>

Co-Authored-By: Claude Opus 4.5 <noreply@anthropic.com>"

# 2. GitHubにプッシュ
git push origin main

# 3. GitHub Pagesにデプロイ
npm run deploy
```

**デプロイ後の確認:**
- 本番URL: https://jime0083.github.io/olive
- デプロイ完了後、上記URLで動作確認を行うこと

### デプロイタイミング

| 作業内容 | デプロイ必須 |
|----------|-------------|
| 新機能の実装 | ✅ |

### チェックリスト

実装完了時に以下を確認：

- [ ] TypeScriptエラーがないこと (`npx tsc --noEmit`)
- [ ] ビルドが成功すること (`npm run build`)
- [ ] GitHubにプッシュ済み
- [ ] GitHub Pagesにデプロイ済み (`npm run deploy`)
- [ ] 本番環境で動作確認済み

### タスク完了の定義（最重要・厳守）

**修正作業が終了しても、タスクは完了扱いにしない。**

各タスクには必ず以下の項目を含めること：
```
- [ ] 修正が完了しているかチェック（ユーザー確認待ち）
```

**タスク完了フロー:**
1. 修正作業を実施
2. **ユーザーの確認を待たずにGitHub Pagesへデプロイ**（コミット・プッシュ・`npm run deploy`）
3. デプロイ完了をユーザーに報告し、本番URLで確認できる旨を伝える
4. ユーザーが本番環境で確認し「問題なし」と指示
5. 「修正が完了しているかチェック」にチェックを入れる
6. タスク完了

⛔ **デプロイ自体は確認を待たずに実行する。ただしタスクを「完了」扱いにするのはユーザーの「問題なし」指示を得てから。**

### 作業前の確認（必須）

**ユーザーから修正指示を受けた際は、必ず以下を実行すること：**

1. 指示内容に不明点がないか確認
2. 不明点がある場合は**作業前に**質問する
3. 不明点が一切ない状態で作業に取り掛かる

⛔ **不明点がある状態で作業を開始してはならない**
